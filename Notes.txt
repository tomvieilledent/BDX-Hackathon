🚀 1. Stack technique (le plus efficace pour vous)

👉 Pour un hackathon, oubliez le natif (trop long).

✅ Recommandé :
Flutter (mobile Android + iOS)
Backend simple :
Firebase (notifications + base de données)
API carto :
Google Maps ou OpenStreetMap

👉 Pourquoi ?

1 seul code
Notifications push faciles
Géolocalisation intégrée
Rapide à déployer en démo
🌍 2. Fonctionnalités clés (MVP hackathon)
📍 1. Géolocalisation utilisateur
Récupérer la position GPS
Détecter si l’utilisateur est dans une zone à risque

👉 Exemple :

Zone inondable
Proximité site SEVESO
Zone feu de forêt
🚨 2. Système d’alertes
Sources possibles :
Météo-France (vigilance météo)
Vigicrues (inondations)
Ministère de la Transition écologique (risques industriels / SEVESO)

👉 Fonctionnement :

Soit via API (si dispo)
Soit via simulation pour la démo hackathon (plus réaliste en temps court)
📢 3. Notifications intelligentes

Exemple :

🚨 “Risque d’inondation dans votre zone”
👉 “Montez en étage, évitez les sous-sols”

📋 4. Checklist dynamique

Très important dans votre idée 👍

✔️ Avant la catastrophe (prévention)
Eau potable
Lampe torche
Radio
Médicaments
Documents importants
⚠️ Pendant la catastrophe (adaptatif)

👉 Exemple inondation :

Couper électricité
Ne pas prendre la voiture
Monter en hauteur

👉 Exemple incendie :

Fermer volets
Arroser abords
Préparer évacuation
🧠 5. Logique intelligente (le cœur de votre app)

Vous pouvez faire un système simple :

SI (position = zone inondable) ET (alerte météo pluie forte)
→ déclencher scénario "inondation"

Puis :

afficher consignes
afficher checklist spécifique
🗺️ 3. Données utiles (France / Bordeaux)

Vous pouvez exploiter :

Cartes de zones à risque (open data)
Plans de prévention des risques (PPR)
Données publiques de Bordeaux Métropole
